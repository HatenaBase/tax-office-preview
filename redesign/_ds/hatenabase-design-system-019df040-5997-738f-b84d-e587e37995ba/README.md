# Hatenabase Design System

Hatenabase, inc.（はてなベース株式会社） の全制作物に適用するデザインシステム。
コーポレートサイト・LP・営業提案資料（Canvaスライド）・チラシ／リーフレットの
すべてで一貫した「コンサルティングファーム風 × ミニマル × 信頼感」のトーンを再現するためのアセット集。

---

## 会社・事業の概要

**Hatenabase, inc.（はてなベース）** は、AIエージェントを活用したバックオフィスDXを専門とするコンサルティング会社。
freee・kintone等のSaaSを「AIの外部記憶」と位置づけ、自律型バックオフィスを構築する。

### 3事業構成

| 事業 | 構成比 | 内容 |
|---|---|---|
| **DX事業** | 50–55% | kintone / Salesforce 導入 + バックオフィスBPR |
| **会計コンサルティング事業** | 30–35% | 経理AX支援 + freee / OBC協業 + 資金調達支援 |
| **研修・人材育成事業** | 10–15% | 有償研修 + 自治体連携 + BPOワーカー育成 |

### 強み
BIG4にはない**開発力**、SIerにはない**会計知見**、現場に入り込む**BPR推進力**。
この3つを内製で持つ唯一のベンダー。

### プロダクト
**ProSet（Professional Setup）** — 経理AX支援の型化ナレッジ × AIエージェント × 育成済みワーカープールのパッケージ。

### 適用先
- コーポレートサイト / LP（サービスLP・研修募集LP・記帳BPO LP 等）
- 営業提案資料（Canvaスライド／15枚以上）
- セミナー告知チラシ・リーフレット（A4 / 三つ折り）

### 事業キーワード（生成時に使える表現）
- 「AIで、バックオフィスを自律させる。」
- 「SaaSを"AIの外部記憶"として活用」

## CONTENT FUNDAMENTALS — 文章のトーンとライティング

### 全体のトーン
**コンサルティングファーム風**：信頼感・知的・断定的。スタートアップ的な軽さや、可愛らしさ・親しみすぎる口語は避ける。
ただし「は**てな**ベース」という社名が示すとおり、「？」を解いていく姿勢のある、語りかけ過ぎない**プロフェッショナル目線**。

### 一人称・呼称
- 自社：「**弊社**」「**Hatenabase**」「**当社**」
- 顧客：「**お客様**」「**御社**」「あなた／みなさん」は使わない
- 二人称：基本は使わない。LPでも「自社の経理は…」のように三人称化する

### 文末・文体
- **である調・断定調**を基本（「〜します」より「〜する」）。ただしLPの本文では「〜します」も可
- 体言止めを多用。`「経理AXの実現。」`のように余白で語る
- 過度な敬語・婉曲表現は避ける（「〜していただけます」より「〜可能」）
- **？** は問題提起・章タイトルでのみ。多用しない

### 漢字／ひらがなのバランス
- 「為に」→「ために」、「事」→「こと」、「物」→「もの」など接続表現はひらがな
- 専門用語・固有名詞は漢字／カタカナ／英字で正確に（freee・kintone・Salesforce は表記揺れ厳禁）
- 助詞は最小限のひらがなで詰める（「〜のためには」より「〜するには」）

### 数字の扱い
- 半角数字・千区切りカンマ（`150社` ではなく **`150社+`** や `1,200件`）
- 実績値は太字 + プライマリカラーで強調（`<strong>150+</strong>` 社）
- パーセントは半角 `%`、年は `2026年`、件数は `件`／`社`

### 略語・カタカナ
- 初出は和文＋括弧で英字フル：「経理AX（Accounting Transformation）」「BPR（Business Process Reengineering）」
- SaaS／DX／AI／BPO は説明不要レベルなので括弧なしで使用
- 「AIエージェント」は英字＋カナ。「エージェント」だけでは曖昧

### 記号
- 句読点：「、」「。」を使用。カンマ・ピリオドは混在させない
- 中黒（・）は並列に積極使用：「DX・AI・会計の融合」
- ダッシュは em ダッシュ「— 」を見出しの副題に：「**ProSet — Professional Setup**」
- 絵文字は**使用しない**

### キャッチコピーの型
1. **問題定義 → 解決の宣言**
   *例：「経理は、まだ手作業に時間を奪われていないか。」 → 「AIで、バックオフィスを自律させる。」*
2. **対比による定義**
   *例：「BIG4にはない開発力、SIerにはない会計知見。」*
3. **メタファー**
   *例：「SaaSを "AIの外部記憶" として活用する。」*

### 禁則
- 「神」「最強」「圧倒的」など過剰煽り表現は使わない
- 「！」は実績数字／チラシのCTA以外では使わない
- 「Let's」「Try」など軽い英字フレーズは使わない（コンサル文脈にそぐわない）

## VISUAL FOUNDATIONS — 視覚の土台

### 色

| 用途 | トークン | HEX |
|---|---|---|
| 見出し・アクセント・CTA | `--color-primary` | `#0A2846`（濃紺） |
| 本文 | `--color-text` | `#191919` |
| サブ本文・キャプション | `--color-text-mute` | `#555B63` |
| 背景 | `--color-bg` | `#FFFFFF` |
| サブ背景／セクション区切り | `--color-bg-sub` | `#F5F7FA` |
| 罫線 | `--color-border` | `#E0E4E8` |

**ルール**
- 多色は使わない。プライマリ濃紺 1色をアクセント／CTA／グラフ／矢印に統一
- グラデーションは控えめに、抽象的なジオメトリック背景にだけ使用（プライマリ濃紺 → 黒）
- 写真は彩度を抑え、青寄り（クール）のトーンに統一
- SaaS ロゴだけは各社ブランドカラーをそのまま使用（色改変禁止）

### 背景・テクスチャ
- 基本は**白背景**。区切りに `#F5F7FA` の薄いグレー帯
- フッターのみ濃紺ベタ + 白文字
- 抽象的なジオメトリック・グラデーション・3Dアイソメトリックを Hero やキービジュアルで活用可
- 手描きイラスト・キャラクター・ソコスト等は**使わない**
- 写真を使う場合は人物 or オフィス／会議の風景。明るく、青寄りの色調補正を統一

### タイポグラフィ
- **Zen Kaku Gothic New**（Google Fonts）/ ウェイト 400・500・700
- フォールバック：`Hiragino Kaku Gothic ProN`, `Noto Sans JP`, sans-serif
- 行間：本文 `1.8`、見出し `1.4–1.5`
- 日本語＋半角英数字を混ぜるとき、英数字も同フォントで統一（カーニング自然）

### 余白・密度
- セクション間 **96px 以上**
- max-width **1200px**、コンテナ余白 24px（モバイル 16px）
- 情報密度は意図的に下げる。1画面1メッセージの原則

### コーナー・矩形
- ボタン：`8px`
- カード：`8–16px`
- スライド図表内：`4px`（鋭く・コンサル的）
- 円形・カプセル形は使わない（チップ・タグだけ例外的に許容）

### 線・罫線
- すべて `1px` `#E0E4E8`、矢印は `2px` `#0A2846`
- 二重線・破線は使わない

### シャドウ
- カード：`0 1px 2px rgba(10,40,70,.04), 0 4px 16px rgba(10,40,70,.06)`（ごく浅い）
- CTA hover：`0 4px 12px rgba(10,40,70,.25)`（プライマリの色相を維持）
- インナーシャドウは使わない

### ボタン・インタラクション状態
- **CTAソリッド**：濃紺塗り＋白文字／角丸 8px
- **アウトライン**：背景透明 + 濃紺 2px ボーダー
- **ホバー**：`transform: translateY(-2px)` + プライマリ色シャドウ
- **プレス**：`transform: translateY(0)` + シャドウ消失
- **テキストCTA**：`→` 矢印付き、ホバーで矢印が 4px 右にスライド
- **リンク（テキスト中）**：下線あり（hover でカラーを `--color-primary-700` に）

### アニメーション
- フェードイン（`opacity` 0→1, 400ms, `--ease-out`）程度に抑える
- パララックス・派手な視差・バウンス・拡大ホバーは使わない
- スクロール起点の reveal は 1セクション 1要素まで

### 透明度・ブラー
- 半透明オーバーレイは Hero 上の文字背景で 0.6–0.8 まで
- backdrop-filter: blur は固定ヘッダー / モーダル背景のみ

### 図表・矢印
- 矩形：角丸 `4px`、塗り `#0A2846`（強調）/ `#F5F7FA`（地）
- 矢印：ストローク `2px`、色 `#0A2846`、矢頭は鋭角の三角
- 表：ヘッダー行 `#0A2846` 背景 + 白文字、セル罫線 `#E0E4E8`

### ロゴ配置の絶対ルール
- **全制作物**（Web・スライド・チラシ）の**右上に Hatenabase ロゴ必須**
- **右下に `©2026 Hatenabase,inc.` 必須**
- SaaS 名がカード／フロー／比較表に登場するときはテキスト隣にロゴ。サイズはコンテナ幅 5–10%。色改変・変形不可

## ICONOGRAPHY — アイコン運用

### 基本方針
**装飾としてのアイコンは最小限**。コンサル風トーンを維持するため、絵文字・キャラクターアイコンは使用しない。

### 採用するアイコン
- **線画フラットアイコン**（線太さ 1.5–2px、塗りなし、`currentColor` 単色）
- 採用ライブラリ：**Lucide**（CDN: `https://unpkg.com/lucide@latest`）— 線が細すぎず、コンサル文脈に馴染むため代替指定
- 単色 `#0A2846` で塗りつぶし、サイズは本文比 1.0–1.25em

> **substitution flag**: リポジトリ／本番テーマには専用アイコンセットが格納されておらず、Font Awesome（`fontawesome/`ディレクトリ）を WordPress テーマで使用していることのみ確認。デザインシステムとしては Lucide を一次採用としたが、Font Awesome に揃えるべきであれば差し替える。**ご指示ください**。

### 採用しないもの
- 絵文字（🚀✅🎯 など） — 一切使用しない
- カラフルな塗り分けアイコン
- カートゥーン・キャラクター
- Unicode 記号アイコン（▶ ★ など） — 図表内の矢印に限り `→` のみ可

### SaaS ロゴ（アイコン代わり）
カード・フロー・比較表での SaaS 言及時はテキストの隣に**公式ロゴ画像**を配置：

| SaaS | ファイル | 公式色 |
|---|---|---|
| freee | `assets/saas-logos/freee.png` (`.svg` 推奨) | `#2864F0` |
| kintone | `assets/saas-logos/kintone.png` | `#F5A623` |
| Salesforce | `assets/saas-logos/salesforce.jpg` | `#00A1E0` |
| バクラク | `assets/saas-logos/bakuraku.png` | `#0052CC` |
| 勘定奉行 | `assets/saas-logos/kanjo-bugyo.png` | `#8B2D8B` |

ロゴは色改変・変形・回転不可。サイズはコンテナ幅 5–10%。

### 自社ロゴ
- `assets/logo/hatenabase-logo.png` — 青の熱気球＋オレンジバスケット＋"Hatenabase"筆記体ワードマーク
- ロゴは単独で使用。背景は白か `#F5F7FA` のみ。濃紺背景時は別途反転版が必要（**未提供 — フラグ**）

## CAVEATS — 既知の制約・要確認事項

- **アイコンセット**：本番 WP テーマは Font Awesome を使用しているが、デザインシステム上の指定は不明。本リポジトリでは Lucide を仮採用（フラグ）
- **濃紺背景用ロゴ反転版**：未提供。現在はフッター内ロゴを CSS `filter: brightness(0) invert(1)` で代用（非推奨）
- **本物のスライド見本（PDF / PPTX）**：Canva マスタ ID のみ提供、画像見本は未提供。本リポジトリの `slides/` はガイドの仕様を元に再現したもの
- **フォント**：Zen Kaku Gothic New を Google Fonts CDN から読み込み。ローカル `.ttf` は同梱していない（オフライン環境では Web から取得不可）
